# Release Checklist

Use this lightweight checklist before publishing a release.

## Pre-release
- [ ] Decide version number (semver).
- [ ] Confirm main is clean and up to date.
- [ ] Verify docs are accurate.
- [ ] Ensure any scripts/examples run as expected.

## Release
- [ ] Create a tag and GitHub release.
- [ ] Summarize changes in release notes.

## Post-release
- [ ] Share the release link.
- [ ] Capture any follow-up tasks as issues.
