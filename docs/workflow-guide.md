# Workflow Guide

A quick, repeatable flow for small contributions.

## Steps
1. Create or pick an issue.
2. Create a branch from `main`.
3. Make a small, meaningful change.
4. Commit with a clear message.
5. Open a PR and link the issue.
6. Merge and delete the branch.

## Tips
- Keep PRs small.
- Update docs when behavior changes.
- Use issue templates.
